## Provenance Fields Verified

All 8 required provenance fields are present in both pages:

1. `templateId` - report template identifier
2. `operator` - who generated the report
3. `sourcePatientSet` - source cohort/patient set
4. `sourceFilters` / `filter_criteria` - applied filter parameters
5. `generatedAt` / `reportTimestamp` - generation timestamp
6. `runId` - unique execution run identifier
7. `sha256_hash` - deterministic content hash
8. `signature` - ed25519 signing signature

## Simulation Mode Behavior Verified

- **Reports page**: Fetches `/api/v1/reports` with 4s timeout. On failure, renders simulation banner with dynamic ISO timestamp and reason ("Backend /api/v1/reports unreachable after 4s timeout").
- **Export page**: Fetches `/api/v1/reports/export/jobs` with 4s timeout. On failure, renders simulation banner with dynamic ISO timestamp and reason. Previously hard-coded to `mode="simulation"`, now dynamic.
- Both pages display `lastUpdatedAt` as the current ISO timestamp when in simulation fallback.
