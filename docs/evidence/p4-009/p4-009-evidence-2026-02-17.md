## 3. Per-Specialty Validation Plan

### 3.1 Coverage Report Generation and Review

For each specialty, generate a coverage report and validate:

| Validation Step | Method | Acceptance Criteria |
|----------------|--------|-------------------|
| Specialty OMOP condition set defined | Extract all OMOP condition concepts for specialty from vocabulary | Complete ICD-10 code list mapped to OMOP concepts |
| Coverage score computed | (# conditions with active guideline) / (# total conditions) | Score meets or exceeds per-specialty target |
| Gap conditions identified | List uncovered OMOP condition concepts | Gaps ranked by prevalence and severity |
| Gap analysis reviewed by domain expert | Expert confirms gap list is accurate and prioritization is correct | Expert sign-off on gap report |

**Per-Specialty Coverage Targets:**

| Specialty | Target Coverage | ICD-10 Scope |
|-----------|----------------|-------------|
| General Internal Medicine | >80% | Chapters A00-B99, D50-D89, E00-E89, G00-G99, I00-I99, J00-J99, K00-K93, M00-M99, N00-N99 |
| Cardiology | >75% | Chapter I (circulatory) + relevant procedure codes |
| Oncology | >60% | Chapter C (neoplasms) + D00-D49 (in situ/benign/uncertain) |
| Nephrology | >85% | N00-N29 (glomerular/renal tubulo-interstitial) + N17-N19 (renal failure) |
| Endocrinology | >70% | E00-E35 (endocrine) + E08-E13 (diabetes) |

### 3.2 Accuracy Validation: Domain Expert Spot-Check

| Parameter | Requirement |
|-----------|------------|
| Sample size per specialty | Minimum 20 guidelines (or 10% of corpus, whichever is greater) |
| Reviewer | Board-appointed domain expert with verified specialty credentials |
| Review dimensions | Clinical accuracy, evidence grade correctness, recommendation completeness, OMOP linkage validity |
| Documentation | Per-guideline spot-check log with pass/fail and comments |
| Threshold | >95% of spot-checked guidelines rated clinically acceptable |
| Remediation | Failed guidelines returned to ingestion pipeline for correction and re-review |

### 3.3 Inter-Rater Reliability Measurement

| Parameter | Requirement |
|-----------|------------|

## Conclusion

Guideline corpus specialty expansion remains correctly deferred per the ADR. The underlying infrastructure is complete: `guideline_rag_service.py` (535 lines) supports multi-specialty retrieval with OMOP hierarchy expansion, and `guideline_version_service.py` (407 lines) provides lifecycle management with staleness detection (P1-012 and P3-013 both closed). The implementation framework defines a complete ingestion pipeline, editorial governance charter, expiry policy, and per-specialty timeline. The validation plan covers editorial board workflow testing (GOV-001 through GOV-008), per-specialty accuracy validation with domain expert spot-checks and inter-rater reliability (kappa >0.8 target), guideline lifecycle testing (LC-001 through LC-007), stale detection testing (SD-001 through SD-007), coverage gap identification (CG-001 through CG-004), and RAG retrieval regression testing (REG-001 through REG-005) to ensure no degradation on general medicine queries after specialty additions. All ten activation gates are currently unmet. Execution begins when the editorial board is formed, domain experts are assigned, and source licensing is cleared -- all organizational prerequisites, not code gaps.
