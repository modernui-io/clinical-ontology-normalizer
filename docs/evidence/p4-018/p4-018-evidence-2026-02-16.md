## 3 Deterministic Scenarios

### Scenario 1: Clinical Safety + Provenance (`p4-018-clinical-safety`)
- **Page:** `/clinical/intelligence`
- **Input:** `{ patientId: "TEST12345", question: "Is the patient at risk for medication interactions?" }`
- **Steps:** 6 (load graph, verify structure, query agent, verify confidence, check guard, verify banner)
- **Endpoints:** `/api/clinical-agent/import`, `/api/clinical-agent/graph/TEST12345`, `/api/clinical-agent/query/TEST12345`

### Scenario 2: Interop + OpenEHR Dry-Run (`p4-018-interop-openehr`)
- **Page:** `/pipelines/openehr/operations`
- **Input:** `{ patientId: "patient-demo-001", composition: "<DEMO_COMPOSITION>" }`
- **Steps:** 6 (load composition, dry-run, verify counts, reconcile, verify fingerprints, check rollback guard)
- **Endpoints:** `/api/v1/openehr/dry-run`, `/api/v1/openehr/reconcile/patient-demo-001`, `/api/v1/openehr/rollback`

### Scenario 3: Quality/Ops Dashboard (`p4-018-quality-ops`)
- **Page:** `/clinical`
- **Input:** `{ dashboardEndpoints: ["/api/dashboard/provider", "/api/dashboard/biller"] }`
- **Steps:** 6 (fetch provider, fetch biller, verify alerts, verify HCC, verify gaps, check guard)
- **Endpoints:** `/api/dashboard/provider`, `/api/dashboard/biller`, `/api/v1/kg/completeness/score`

## DemoScenarioRunner Features

- Collapsible `<details>` with `id="demo-scenario-runner"` for deep-linking from sales-demo
- Operator name input (required before run)
- Sequential step execution with 300ms visual delay
- Each step: probes real API with 3s timeout, records hit vs simulated
- Step log: vertical timeline with timestamp, status icon, endpoint, duration
- Endpoint summary: "Hit (Live)" vs "Simulated" columns
- Evidence manifest JSON export with: scenarioId, inputHash, stepResults, endpointsHit/Simulated, operator, timestamps, mode
- Simulation mode: amber banner blocking write actions with escalation text
- 3-click reviewer path: Navigate to page -> Click "Run Scenario" -> Click "Export Evidence"
