## API Key Lifecycle Test Plan

### Test: Create -> Rotate -> Revoke -> Audit

| Step | Action | Expected Result | Pass Criteria |
|------|--------|----------------|---------------|
| 1.1 | Create API key for tenant | Key returned (plaintext shown once) | key_prefix present, key_hash stored |
| 1.2 | Authenticate with new key | 200 OK on test endpoint | Request succeeds with tenant context |
| 1.3 | Verify key metadata | Query key details by ID | All schema fields populated |
| 1.4 | Initiate key rotation | New key generated, old key enters grace period | Both keys work during 24-hour window |
| 1.5 | Wait for grace period expiry | Old key expires | Old key returns 401 |
| 1.6 | Verify new key works | Test endpoint with rotated key | 200 OK |
| 1.7 | Revoke key | Key marked revoked | Immediate 401 on next request |
| 1.8 | Verify audit trail | Query audit log for key lifecycle | Create, rotate, revoke events all logged |
| 1.9 | Attempt reuse of revoked key | Send request with revoked key | 401 Unauthorized |
| 1.10 | Verify key never stored as plaintext | Database inspection | Only key_hash present, no plaintext |

## Gap Analysis Against Existing Services

### `rbac_service.py` (1,013 lines)
- [x] Role-based access control implemented — foundation for API key scoping
- [x] Permission evaluation per request — compatible with scope-based API keys
- [x] Tenant-aware authorization — tenant isolation already enforced
- [ ] **Gap:** No API key authentication flow — requires new auth provider alongside existing JWT/session
- [ ] **Gap:** No scope-based permission model — requires scope-to-RBAC-permission mapping
- [ ] **Gap:** No rate limit integration — requires middleware addition

### `tenant.py` (260 lines)
- [x] Tenant boundary enforcement at query level — foundation for API key tenant scoping
- [x] Tenant context propagation — already threads through middleware
- [ ] **Gap:** No external tenant provisioning API — requires sandbox provisioning endpoint
- [ ] **Gap:** No synthetic data seeding per tenant — requires data fixture loader

### `audit_middleware.py` (630 lines)
- [x] Request-level audit logging — foundation for API usage telemetry
- [x] Tenant context in audit events — already captures tenant_id
- [ ] **Gap:** No API key identification in audit events — requires key_id field addition
- [ ] **Gap:** No usage aggregation pipeline — requires telemetry rollup jobs

### `test_openapi_spec.py` (141 lines)
- [x] OpenAPI spec validation — foundation for developer portal
- [x] Tag, example, and description checks — quality baseline for external docs
- [ ] **Gap:** No external vs internal endpoint classification — requires tagging/filtering
- [ ] **Gap:** No SDK generation pipeline — requires OpenAPI codegen integration
