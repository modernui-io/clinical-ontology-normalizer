## Consumer Overhead Measurement Plan

### Ops Burden Delta: Monolith vs. Mesh

| Metric | Monolith (Current) | Mesh (Projected) | Measurement Method |
|--------|--------------------|--------------------|-------------------|
| Backup jobs | 1 (single PostgreSQL) | N (per domain) | Count of scheduled backup jobs |
| Monitoring dashboards | 1 (unified) | N+1 (per domain + global) | Dashboard count |
| On-call rotations | 1 (shared) | N (per domain) | Rotation schedule count |
| Schema migration complexity | Low (single DB) | Medium (per-domain migrations) | Migration time delta |
| Cross-domain query latency | <10ms (single JOIN) | 50-500ms (API calls) | p95 latency measurement |
| Incident response | Single DB troubleshooting | Multi-domain root cause analysis | MTTR comparison |
| Total FTE for data ops | ~0.5 FTE | ~1.5-2.0 FTE (estimated) | Time tracking |

### Measurement Protocol
1. Baseline current ops burden (2-week observation window)
2. Deploy analytics domain pilot
3. Measure pilot ops burden (4-week observation window)
4. Compute delta per metric
5. Extrapolate to full mesh (4 domains)
6. Decision: proceed with mesh if total ops FTE < 2x baseline AND cross-domain latency < 500ms p95
