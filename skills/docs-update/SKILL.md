# Skill: Docs Update

## When to Use
- Update docs, maps, or guidance after a change.

## Workflow
1. Update primary doc (e.g., `README.md`, `CODEBASE_MAP.md`, `DEVELOPMENT.md`).
2. Keep examples minimal and accurate.
3. If the change affects architecture or flow, update `CODEBASE_MAP.md`.
4. If the change affects agent guidance, update `AGENTS.md`.

## Notes
- Keep docs concise and actionable.
